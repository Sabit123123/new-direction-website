# Настройка и что нового

## Что добавлено в планировщик (`planner.html`)

1. **Каталог из реального инвентаря** — ~145 позиций из «Список оборудования.xlsx»
   (звук, свет, лазеры, видео, металлоконструкции, бэклайн, DJ, спецэффекты, питание) + базовая мебель.
   Данные вынесены в `catalog.js`.
2. **3D-модели по размерам** — 54 реальные модели (Kenney CC0 + Poly Pizza), остальное —
   детальная процедурная генерация по точным габаритам. См. `MODELS.md`.
3. **Импорт моделей** в форматах **GLB / GLTF / OBJ / DAE / FBX / STL** — кнопка «Загрузить модель»
   в форме «Новая позиция» и в свойствах выбранного объекта. Модель авто-масштабируется под размеры.
4. **Два списка тегов**: **Вид** (свет/звук/экран/…) и **Комплект** (обычный/средний/дорогой/бизнес/ультра) —
   фильтры каталога, выбор при создании позиции и редактирование у размещённого объекта.
5. **Новая позиция** — кнопка «+ Новая позиция»: название, вид, комплект, размеры, цвет, своя 3D-модель.
   Сохраняется в браузере (localStorage).
6. **Сохранение сцен** с названием, адресом, ссылками **Google Maps** и **2ГИС**, превью и
   галочкой **«Показывать на основном сайте»** → публикация в ленту.

## Лента планировок на главной (`index.html`)

Опубликованные сцены появляются в новой секции **«Планировки залов»** (пункт меню «Планировки»)

## Настройка Firebase (нужна один раз, чтобы лента работала для всех посетителей)

Пока `firebase-config.js` не заполнен — сайт работает как обычно, лента просто скрыта/пустая.

1. Зайдите на **console.firebase.google.com** → создайте бесплатный проект.
2. **Build → Firestore Database → Create database** (Production mode, регион ближе к вам).
3. **Build → Authentication → Sign-in method** → включите **Email/Password**.
   Во вкладке **Users** добавьте администратора (e-mail + пароль) — этим логином вы будете публиковать.
   Скопируйте **User UID** этого пользователя.
4. **Project settings (⚙) → Your apps → Web (</>)** → зарегистрируйте веб-приложение и
   скопируйте объект `firebaseConfig`. Вставьте значения в `firebase-config.js` (верх файла).
5. **Firestore → Rules** → вставьте (замените `ADMIN_UID` на UID из шага 3):
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /scenes/{id} {
         allow read: if true;
         allow write: if request.auth != null && request.auth.uid == 'ADMIN_UID';
       }
     }
   }
   ```
6. Готово. В планировщике: **Сохранить → «Показывать на основном сайте»** → введите e-mail/пароль
   администратора → сцена публикуется и появляется в ленте на `index.html`.

## Локальный запуск

Это статические файлы — откройте через любой статический сервер из корня репозитория
(модели грузятся по HTTP, поэтому `file://` не подойдёт). Например: `npx serve` или `npm run dev`.

> Примечание: Draco-сжатые glTF не поддерживаются (нет DRACOLoader).
